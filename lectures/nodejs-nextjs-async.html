What is an HTTP Server?

A program that:

1 Listens on a network port (usually 80 or 443)
2 Receives HTTP requests from clients
3 Processes the request (routing, logic, DB)
4 Sends back an HTTP response

The net Module -- Raw TCP

TCP Server
const net = require('net');

const server = net.createServer((socket) => {
    console.log('Client connected');

    socket.on('data', (data) => {
        console.log('Received:', data.toString());
        socket.write('Echo: ' + data);
    });

    socket.on('end', () => {
        console.log('Client disconnected');
    });
});

server.listen(8080, () => {
    console.log('TCP server on port 8080');
});
TCP Client
const net = require('net');

const client = net.createConnection({
    port: 8080
}, () => {
    console.log('Connected to server');
    client.write('Hello, server!');
});

client.on('data', (data) => {
    console.log('Server says:', data.toString());
    client.end();
});

client.on('end', () => {
    console.log('Disconnected');
});
Use cases: Custom protocols, game servers, chat applications, IoT

Blocking vs Non-Blocking I/O

Blocking (BAD)
const fs = require('fs');

// Thread STOPS here until file is read
const data = fs.readFileSync('big-file.txt');
console.log(data);

// Nothing else runs during file read
// Server can't handle other requests!
Blocks entire event loop. No other requests handled.
Non-Blocking (GOOD)
const fs = require('fs');

// Returns immediately, callback runs later
fs.readFile('big-file.txt', (err, data) => {
    if (err) throw err;
    console.log(data);
});

// This runs while file is being read!
console.log('Reading file...');
Event loop stays free. Server handles other requests.

Your First Node.js HTTP Server

server.js
const http = require('http');

const server = http.createServer((req, res) => {
    // req = incoming request (method, url, headers)
    // res = outgoing response (write status, headers, body)

    if (req.method === 'GET' && req.url === '/api/hello') {
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ message: 'Hello, World!' }));
    } else {
        res.writeHead(404);
        res.end('Not Found');
    }
});

server.listen(3000, () => {
    console.log('Server running at http://localhost:3000');
});
node server.js

API Routes -- Backend in Next.js

app/api/users/route.ts
import { NextResponse } from 'next/server';
import { db } from '@/lib/db';

// GET /api/users
export async function GET(request: Request) {
    const users = await db.user.findMany();
    return NextResponse.json(users);
}

// POST /api/users
export async function POST(request: Request) {
    const body = await request.json();

    const user = await db.user.create({
        data: { name: body.name, email: body.email }
    });

    return NextResponse.json(user, { status: 201 });
}

// Dynamic route: app/api/users/[id]/route.ts
export async function GET(
    request: Request,
    { params }: { params: { id: string } }
) {
    const user = await db.user.findUnique({ where: { id: params.id } });
    if (!user) return NextResponse.json({ error: 'Not found' }, { status: 404 });
    return NextResponse.json(user);
}

The Callback Problem

Callback Hell
getUser(userId, (err, user) => {
    if (err) return handleError(err);

    getOrders(user.id, (err, orders) => {
        if (err) return handleError(err);

        getOrderDetails(orders[0].id, (err, details) => {
            if (err) return handleError(err);

            getShippingStatus(details.shipmentId, (err, status) => {
                if (err) return handleError(err);

                // Finally! But this is unreadable...
                console.log(status);
            });
        });
    });
});
Hard to read -- pyramid of doom Hard to debug -- error handling scattered Hard to sequence -- parallel? series? race?

What is a Promise?

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation.

[?]
Pending
Initial state. Operation in progress.
->
[OK]
Fulfilled
Operation completed successfully.
[X]
Rejected
Operation failed with an error.

Creating Promises

Manual Promise
function delay(ms) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve(`Done after ${ms}ms`);
        }, ms);
    });
}

// Usage
delay(1000).then(result => {
    console.log(result);
});
Promisifying Callbacks
const fs = require('fs');
const { promisify } = require('util');

// Old callback-based
fs.readFile('file.txt', callback);

// Promisified version
const readFileAsync = promisify(fs.readFile);
readFileAsync('file.txt')
    .then(data => console.log(data));

// Or use fs.promises (modern Node)
const { readFile } = require('fs/promises');
readFile('file.txt').then(...);

Promise Chaining

Flat, readable, chainable
getUser(userId)
    .then(user => {
        console.log('Got user:', user.name);
        return getOrders(user.id);  // Return promise for next .then()
    })
    .then(orders => {
        console.log('Got orders:', orders.length);
        return getOrderDetails(orders[0].id);
    })
    .then(details => {
        console.log('Got details:', details);
        return getShippingStatus(details.shipmentId);
    })
    .then(status => {
        console.log('Shipping status:', status);
    })
    .catch(err => {
        // One error handler for the entire chain!
        console.error('Error:', err.message);
    });
Each .then() returns a new Promise. Errors bubble to .catch().

Promise Combinators

Click any combinator for details

Promise.all()

Wait for ALL promises. Fail fast on first rejection.

const [users, posts] = await Promise.all([
    fetch('/api/users'),
    fetch('/api/posts')
]);

Promise.allSettled()

Wait for ALL, never rejects. Get status of each.

const results = await Promise.allSettled([
    fetch('/api/a'),
    fetch('/api/b')  // Even if this fails...
]);
// [{status:'fulfilled',value:...}, {status:'rejected',reason:...}]

Promise.race()

First to settle (fulfill OR reject) wins.

const result = await Promise.race([
    fetch('/api/data'),
    delay(5000).then(() => { throw new Error('Timeout') })
]);

Promise.any()

First to FULFILL wins. Only rejects if all reject.

const fastest = await Promise.any([
    fetch('https://cdn1.example.com/data'),
    fetch('https://cdn2.example.com/data'),
    fetch('https://cdn3.example.com/data')
]);

async/await -- Promises Made Beautiful

Promise Chain
function fetchUserData(id) {
    return getUser(id)
        .then(user => getOrders(user.id))
        .then(orders => {
            return getOrderDetails(orders[0].id);
        })
        .then(details => {
            return details;
        })
        .catch(err => {
            console.error(err);
            throw err;
        });
}
async/await
async function fetchUserData(id) {
    try {
        const user = await getUser(id);
        const orders = await getOrders(user.id);
        const details = await getOrderDetails(
            orders[0].id
        );
        return details;
    } catch (err) {
        console.error(err);
        throw err;
    }
}
async makes function return a Promise await pauses until Promise settles Use try/catch for errors

Common Async Patterns

Click any pattern for details

Sequential

One after another. Total time = sum of all.

const a = await fetchA();  // 1s
const b = await fetchB();  // 1s
// Total: 2 seconds

Parallel

All at once. Total time = slowest.

const [a, b] = await Promise.all([
    fetchA(),  // 1s
    fetchB()   // 1s
]);
// Total: 1 second!

Sequential Loop

Process items one by one.

for (const id of ids) {
    await processItem(id);
    // Each waits for previous
}

Parallel Loop

Process all items concurrently.

await Promise.all(
    ids.map(id => processItem(id))
);
// All run simultaneously

Error Handling Best Practices

Don't
// Unhandled promise rejection!
async function bad() {
    const data = await fetch('/api/data');
    return data.json();
}

// Error silently swallowed
promise.catch(() => {});

// No error handling at all
fetch('/api').then(res => res.json());
Do
async function good() {
    try {
        const res = await fetch('/api/data');
        if (!res.ok) {
            throw new Error(`HTTP ${res.status}`);
        }
        return res.json();
    } catch (err) {
        console.error('Fetch failed:', err);
        throw err;  // Re-throw or handle
    }
}

// Or use .catch() for simple cases
const data = await fetchData().catch(handleError);

Async Gotchas

Click any gotcha for detailed explanation

forEach doesn't await

// WRONG - fires all at once, doesn't wait
items.forEach(async (item) => {
    await processItem(item);
});

// RIGHT - use for...of
for (const item of items) {
    await processItem(item);
}

Await in wrong place

// WRONG - sequential when could be parallel
const a = await fetchA();
const b = await fetchB();

// RIGHT - parallel with Promise.all
const [a, b] = await Promise.all([
    fetchA(), fetchB()
]);

Missing await

async function save(data) {
    // WRONG - returns promise, not result
    return db.save(data);

    // RIGHT
    return await db.save(data);
    // Or let async handle it (same effect)
}

Blocking the event loop

// WRONG - CPU-bound work blocks
for (let i = 0; i < 1e9; i++) { /* calc */ }

// RIGHT - use worker threads for CPU work
const { Worker } = require('worker_threads');
new Worker('./heavy-calc.js');

The Problem React Solves

Vanilla JavaScript

// Manual DOM manipulation
const button = document.getElementById('like-btn');
const counter = document.getElementById('count');
let likes = 0;

button.addEventListener('click', () => {
    likes++;
    counter.textContent = likes;
    // What if count appears elsewhere?
    // What if we need to sync with server?
    // What about undo? Animation? Loading state?
    // Manual updates for EVERY state change!
});
Manual DOM updates State scattered everywhere Hard to reason about

React Way

function LikeButton() {
    const [likes, setLikes] = useState(0);

    return (
        <button onClick={() => setLikes(likes + 1)}>
            {likes} Likes
        </button>
    );
}

// State changes? React updates the DOM for you.
// Need it elsewhere? Just use the same state.
// UI is a function of state. Always.
Declarative updates State in one place Predictable UI

Component Rules

  • Name starts with capital — Welcome not welcome
  • Returns JSX — Or null for nothing
  • Pure function — Same props = same output
  • One root element — Or use Fragment <>...</>

When to Create a Component?

  • Reusable piece (Button, Card, Modal)
  • Logical separation (Header, Sidebar)
  • Complex section (UserProfile, OrderList)

useState — Managing State

Basic Usage
import { useState } from 'react';

function Counter() {
    // [currentValue, setterFunction] = useState(initialValue)
    const [count, setCount] = useState(0);

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
            <button onClick={() => setCount(0)}>
                Reset
            </button>
        </div>
    );
}
Functional Updates
// When new state depends on previous state,
// use functional update form:

function Counter() {
    const [count, setCount] = useState(0);

    const incrementThree = () => {
        // Wrong - all use same stale count
        // setCount(count + 1);
        // setCount(count + 1);
        // setCount(count + 1);

        // Correct - each uses latest value
        setCount(prev => prev + 1);
        setCount(prev => prev + 1);
        setCount(prev => prev + 1);
    };
}

useState with Objects & Arrays

Wrong - Mutation
const [user, setUser] = useState({
    name: 'John',
    age: 25
});

// WRONG - mutating state directly
user.age = 26;
setUser(user); // React won't re-render!

// WRONG - same object reference
const updateAge = () => {
    user.age = 26;
    setUser(user);
};
Correct - New Object
const [user, setUser] = useState({
    name: 'John',
    age: 25
});

// CORRECT - create new object
setUser({ ...user, age: 26 });

// CORRECT - with functional update
setUser(prev => ({ ...prev, age: prev.age + 1 }));

// Arrays - same principle
const [items, setItems] = useState([]);
setItems([...items, newItem]);     // Add
setItems(items.filter(i => i.id !== id)); // Remove
Immutability: Always create new objects/arrays. React detects changes by reference comparison.

useEffect — Side Effects

useEffect Patterns
import { useEffect, useState } from 'react';

function UserProfile({ userId }) {
    const [user, setUser] = useState(null);

    // Run on mount and when userId changes
    useEffect(() => {
        async function fetchUser() {
            const response = await fetch(`/api/users/${userId}`);
            const data = await response.json();
            setUser(data);
        }
        fetchUser();
    }, [userId]); // Dependency array

    // Cleanup function (runs before next effect or unmount)
    useEffect(() => {
        const subscription = subscribeToUser(userId);

        return () => {
            subscription.unsubscribe(); // Cleanup!
        };
    }, [userId]);

    if (!user) return <Loading />;
    return <div>{user.name}</div>;
}
[] = run once on mount [dep] = run when dep changes no array = run every render (rare!)

useEffect Gotchas

Missing Dependencies

// Bug: count is stale
useEffect(() => {
    setInterval(() => {
        console.log(count); // Always 0!
    }, 1000);
}, []); // count missing from deps

Infinite Loop

// Bug: object recreated every render
useEffect(() => {
    fetchData(options);
}, [options]); // options = {} each time!

// Fix: useMemo or move inside effect

Race Condition

// Bug: stale response overwrites fresh
useEffect(() => {
    let cancelled = false;
    fetchUser(id).then(data => {
        if (!cancelled) setUser(data);
    });
    return () => { cancelled = true; };
}, [id]);
Tip: Use eslint-plugin-react-hooks — it catches most issues.

Custom Hooks — Reusable Logic

useLocalStorage Hook
function useLocalStorage(key, initialValue) {
    const [value, setValue] = useState(() => {
        const stored = localStorage.getItem(key);
        return stored ? JSON.parse(stored) : initialValue;
    });

    useEffect(() => {
        localStorage.setItem(key, JSON.stringify(value));
    }, [key, value]);

    return [value, setValue];
}

// Usage:
const [theme, setTheme] = useLocalStorage('theme', 'dark');
useFetch Hook
function useFetch(url) {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);

    useEffect(() => {
        setLoading(true);
        fetch(url)
            .then(res => res.json())
            .then(setData)
            .catch(setError)
            .finally(() => setLoading(false));
    }, [url]);

    return { data, loading, error };
}

// Usage:
const { data, loading } = useFetch('/api/users');
Rule: Custom hooks must start with use — this enables the linter to check hook rules.

    <Layout user={user}>
        <Sidebar user={user}>

                <UserAvatar user={user} />  {/* Finally uses it! */}
            </NavMenu>
        </Sidebar>
    </Layout>

Tedious to maintain Components tightly coupled Hard to refactor

Context API — The Solution

Creating and Using Context
// 1. Create context
const UserContext = createContext(null);

// 2. Create provider component
function UserProvider({ children }) {
    const [user, setUser] = useState(null);

    const login = async (credentials) => {
        const user = await authApi.login(credentials);
        setUser(user);
    };

    const logout = () => setUser(null);

    return (
        <UserContext.Provider value={{ user, login, logout }}>
            {children}
        </UserContext.Provider>
    );
}

// 3. Use in any nested component
function UserAvatar() {
    const { user } = useContext(UserContext);
    return <img src={user?.avatar} alt={user?.name} />;
}

// 4. Wrap app with provider
<UserProvider>
    <App />  {/* Any component inside can useContext */}
</UserProvider>

Context Best Practices

Split Contexts by Domain

// Don't: One giant context
<AppContext.Provider value={{ user, theme, cart, ... }}>

// Do: Separate contexts
<UserProvider>
    <ThemeProvider>
        <CartProvider>
            <App />
        </CartProvider>
    </ThemeProvider>
</UserProvider>

Create Custom Hook

// Cleaner API than raw useContext
function useUser() {
    const context = useContext(UserContext);
    if (!context) {
        throw new Error('useUser must be within UserProvider');
    }
    return context;
}

// Usage
const { user, login } = useUser();
Performance Note: Context changes re-render all consumers. For frequent updates (e.g., mouse position), consider other solutions.

Making React Fast

How React Renders

1

Trigger

State change, props change, or parent re-renders

→
2

Render

React calls your component function

→
3

Reconcile

Compare new tree with previous (diffing)

→
4

Commit

Apply necessary DOM updates

Key insight: "Render" means calling the function. It doesn't necessarily mean DOM updates. React is smart about only updating what changed.

React.memo — Skip Unnecessary Renders

Without memo
function ExpensiveList({ items }) {
    console.log('Rendering list...');
    return items.map(item => (
        <div key={item.id}>{item.name}</div>
    ));
}

function Parent() {
    const [count, setCount] = useState(0);
    const items = [/* ... */];

    return (
        <div>
            <button onClick={() => setCount(c => c + 1)}>
                Count: {count}
            </button>
            {/* Re-renders even though items didn't change! */}
            <ExpensiveList items={items} />
        </div>
    );
}
With memo
const ExpensiveList = memo(function ExpensiveList({ items }) {
    console.log('Rendering list...');
    return items.map(item => (
        <div key={item.id}>{item.name}</div>
    ));
});

// Now only re-renders if items actually change
// (shallow comparison of props)

// For deep comparison:
const MemoizedComponent = memo(Component, (prev, next) => {
    return prev.id === next.id; // Custom comparison
});
Don't over-use: memo has overhead. Profile first!

useMemo & useCallback

useMemo — Cache Values
function FilteredList({ items, filter }) {
    // Without useMemo: runs every render
    // const filtered = items.filter(expensiveFilter);

    // With useMemo: only when deps change
    const filtered = useMemo(() => {
        return items.filter(item =>
            expensiveFilter(item, filter)
        );
    }, [items, filter]);

    return <List items={filtered} />;
}
useCallback — Cache Functions
function Parent() {
    const [count, setCount] = useState(0);

    // Without useCallback: new function every render
    // const handleClick = () => { ... };

    // With useCallback: stable reference
    const handleClick = useCallback(() => {
        console.log('Clicked!');
    }, []); // Empty deps = never changes

    // Important when passing to memoized children
    return <MemoizedChild onClick={handleClick} />;
}
Rule: Don't optimize prematurely. Measure with React DevTools Profiler first.

Code Splitting with lazy()

Lazy Loading Components
import { lazy, Suspense } from 'react';

// Instead of: import HeavyChart from './HeavyChart';
const HeavyChart = lazy(() => import('./HeavyChart'));
const AdminPanel = lazy(() => import('./AdminPanel'));

function App() {
    return (
        <div>
            <Header />

            {/* Suspense shows fallback while loading */}
            <Suspense fallback={<Spinner />}>
                <HeavyChart data={data} />
            </Suspense>

            {/* Route-based splitting */}
            <Routes>
                <Route path="/admin" element={
                    <Suspense fallback={<Loading />}>
                        <AdminPanel />
                    </Suspense>
                } />
            </Routes>
        </div>
    );
}
Benefit: Initial bundle smaller → faster page load. Components load on-demand.

React Server Components

Server Components (default in Next.js 13+)

  • Render on the server only
  • Can access database, filesystem directly
  • Zero JS sent to client
  • Can't use hooks or browser APIs
// app/users/page.tsx (Server Component)
async function UsersPage() {
    const users = await db.query('SELECT * FROM users');
    return <UserList users={users} />;
}

Client Components ('use client')

  • Run in the browser
  • Can use hooks, events, browser APIs
  • JS shipped to client
'use client';

function LikeButton() {
    const [likes, setLikes] = useState(0);
    return <button onClick={() => setLikes(l => l + 1)}>...</button>;
}

Error Boundaries

ErrorBoundary.tsx
class ErrorBoundary extends React.Component {
    state = { hasError: false, error: null };

    static getDerivedStateFromError(error) {
        return { hasError: true, error };
    }

    componentDidCatch(error, errorInfo) {
        // Log to error tracking service (Sentry, etc.)
        logErrorToService(error, errorInfo);
    }

    render() {
        if (this.state.hasError) {
            return (
                <div className="error-fallback">
                    <h2>Something went wrong</h2>
                    <button onClick={() => this.setState({ hasError: false })}>
                        Try again
                    </button>
                </div>
            );
        }
        return this.props.children;
    }
}

// Usage
<ErrorBoundary>
    <MyComponent />  {/* If this crashes, fallback shows */}
</ErrorBoundary>

Key Takeaways

01
Think in components

UI = f(state). Components are functions that return UI based on their props and state.

02
State is immutable

Never mutate state directly. Always create new objects/arrays.

03
Effects have cleanup

Return a cleanup function from useEffect. Watch your dependencies.

04
Optimize with measurement

Don't guess. Profile first, then apply memo/useMemo/useCallback where needed.

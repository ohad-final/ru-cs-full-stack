Why TypeScript?

The benefits that changed everything

TypeScript = JavaScript + Types

TypeScript Benefits

[1]

Catch Errors Early

Find bugs at compile time, not runtime. No more "undefined is not a function".

[2]

Better IDE Support

Autocomplete, inline docs, go-to-definition, find all references.

[3]

Self-Documenting

Types ARE documentation. Always up-to-date, never lies.

[4]

Fearless Refactoring

Rename, restructure, delete — compiler catches all the breakage.

[5]

Team Collaboration

Clear contracts between modules. New team members onboard faster.

[6]

It's Still JavaScript

100% compatible. Use any JS library. Gradual adoption possible.

Catching Bugs Before They Happen

TypeScript catches these at compile time
interface Product {
    id: number;
    name: string;
    price: number;
}

function displayProduct(product: Product) {
    // Typo - caught immediately!
    console.log(product.naem);
    //                  ~~~~ Property 'naem' does not exist

    // Wrong type - caught!
    const doubled = product.price * "2";
    //                              ~~~ Argument of type 'string'...

    // Missing property - caught!
    const newProduct: Product = { id: 1, name: "Widget" };
    //    ~~~~~~~~~~ Property 'price' is missing

    // Null check - caught!
    const items: Product[] | null = null;
    console.log(items.length);
    //          ~~~~~ Object is possibly 'null'
}

TypeScript Fundamentals

The essential building blocks

const message: string = "Hello, TypeScript!";

Basic Types

Primitives

// String
let name: string = "Alice";
let template: string = `Hello, ${name}`;

// Number (no int/float distinction)
let age: number = 25;
let price: number = 99.99;
let hex: number = 0xff;

// Boolean
let isActive: boolean = true;
let hasAccess: boolean = false;

// Null & Undefined
let nothing: null = null;
let notDefined: undefined = undefined;

Special Types

// Any - escape hatch (avoid!)
let flexible: any = "can be anything";
flexible = 42;  // no error

// Unknown - safer than any
let input: unknown = getInput();
// Must narrow before use
if (typeof input === "string") {
    console.log(input.toUpperCase());
}

// Void - no return value
function log(msg: string): void {
    console.log(msg);
}

// Never - never returns
function fail(msg: string): never {
    throw new Error(msg);
}

Arrays & Tuples

Arrays
// Two syntaxes (identical)
let numbers: number[] = [1, 2, 3];
let strings: Array = ["a", "b"];

// Array of objects
let users: { name: string }[] = [
    { name: "Alice" },
    { name: "Bob" }
];

// Mixed arrays need union types
let mixed: (string | number)[] = [1, "two", 3];

// Readonly arrays
const frozen: readonly number[] = [1, 2, 3];
frozen.push(4);  // Error!
Tuples
// Fixed-length array with specific types
let point: [number, number] = [10, 20];

// Named tuples (TS 4.0+)
type Point = [x: number, y: number];
let coord: Point = [100, 200];

// Common use: React useState
const [count, setCount] = useState(0);
// Type: [number, (n: number) => void]

// Rest elements in tuples
type StringNumberBooleans =
    [string, number, ...boolean[]];
let snb: StringNumberBooleans =
    ["hello", 42, true, false, true];

Objects & Interfaces

Defining object shapes
// Inline object type
let user: { name: string; age: number } = {
    name: "Alice",
    age: 30
};

// Interface - preferred for objects
interface User {
    id: number;
    name: string;
    email: string;
    age?: number;           // Optional property
    readonly createdAt: Date; // Can't be changed
}

const alice: User = {
    id: 1,
    name: "Alice",
    email: "alice@example.com",
    createdAt: new Date()
};

// Extending interfaces
interface Admin extends User {
    permissions: string[];
}

const admin: Admin = {
    ...alice,
    permissions: ["read", "write", "delete"]
};

Type Inference — Let TypeScript Do the Work

Explicit (verbose)
// You CAN write all types...
const name: string = "Alice";
const age: number = 30;
const active: boolean = true;

const numbers: number[] = [1, 2, 3];

const user: { name: string; age: number } = {
    name: "Alice",
    age: 30
};

function double(n: number): number {
    return n * 2;
}
Inferred (preferred)
// But TypeScript infers most types!
const name = "Alice";        // string
const age = 30;              // number
const active = true;         // boolean

const numbers = [1, 2, 3];   // number[]

const user = {
    name: "Alice",           // { name: string;
    age: 30                  //   age: number }
};

function double(n: number) {
    return n * 2;            // returns number
}
Rule of thumb: Add types for function parameters. Let TypeScript infer the rest.

Union Types & Literal Types

Combining types
// Union: value can be one of several types
let id: string | number;
id = "abc123";  // OK
id = 42;        // OK
id = true;      // Error!

// Literal types: exact values
let direction: "north" | "south" | "east" | "west";
direction = "north";  // OK
direction = "up";     // Error!

// Combining with union
type Status = "pending" | "approved" | "rejected";
type StatusCode = 200 | 201 | 400 | 404 | 500;

// Powerful pattern: discriminated unions
type Result =
    | { success: true; data: T }
    | { success: false; error: string };

function handleResult(result: Result) {
    if (result.success) {
        console.log(result.data.name);  // TypeScript knows data exists
    } else {
        console.log(result.error);       // TypeScript knows error exists
    }
}

Type Aliases

Creating reusable types
// Simple alias
type ID = string | number;
type Username = string;

// Object type alias
type Point = {
    x: number;
    y: number;
};

// Function type alias
type Callback = (data: string) => void;
type AsyncCallback = (data: string) => Promise;

// Generic type alias
type Container = {
    value: T;
    timestamp: Date;
};

const stringContainer: Container = {
    value: "hello",
    timestamp: new Date()
};

Type Alias vs Interface

Use Interface for:
  • Object shapes
  • Class contracts
  • Extending/implementing
Use Type for:
  • Unions & intersections
  • Primitives & tuples
  • Function signatures
  • Mapped types
Tip: When in doubt, use interface for objects, type for everything else.

Functions with Types

Typing function parameters and returns
// Basic function with types
function add(a: number, b: number): number {
    return a + b;
}

// Arrow function
const multiply = (a: number, b: number): number => a * b;

// Optional parameters
function greet(name: string, greeting?: string): string {
    return `${greeting ?? "Hello"}, ${name}!`;
}
greet("Alice");           // "Hello, Alice!"
greet("Alice", "Hi");     // "Hi, Alice!"

// Default parameters
function createUser(name: string, role: string = "user") {
    return { name, role };
}

// Rest parameters
function sum(...numbers: number[]): number {
    return numbers.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4);  // 10

// Function overloads
function format(value: string): string;
function format(value: number): string;
function format(value: string | number): string {
    return String(value);
}

Generics — Reusable Type Logic

Type parameters for flexible, type-safe code
// Without generics: lose type info or repeat yourself
function identityAny(value: any): any { return value; }
function identityString(value: string): string { return value; }
function identityNumber(value: number): number { return value; }

// With generics: one function, all types preserved
function identity(value: T): T {
    return value;
}
identity("hello");  // returns string
identity(42);       // returns number
identity({ x: 1 }); // returns { x: number }

// Generic interfaces
interface ApiResponse {
    data: T;
    status: number;
    error?: string;
}

const userResponse: ApiResponse = {
    data: { id: 1, name: "Alice", email: "..." },
    status: 200
};

// Generic constraints
function getLength(item: T): number {
    return item.length;
}
getLength("hello");     // OK - string has length
getLength([1, 2, 3]);   // OK - array has length
getLength(42);          // Error - number has no length

Type Guards & Narrowing

Narrowing union types at runtime
// TypeScript narrows types based on checks
function process(value: string | number) {
    // typeof guard
    if (typeof value === "string") {
        console.log(value.toUpperCase());  // TypeScript knows it's string
    } else {
        console.log(value.toFixed(2));     // TypeScript knows it's number
    }
}

// instanceof guard
function logDate(value: Date | string) {
    if (value instanceof Date) {
        console.log(value.toISOString());  // It's a Date
    } else {
        console.log(value);                // It's a string
    }
}

// "in" operator guard
type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird) {
    if ("swim" in animal) {
        animal.swim();  // TypeScript knows it's Fish
    } else {
        animal.fly();   // TypeScript knows it's Bird
    }
}

// Custom type guard
function isString(value: unknown): value is string {
    return typeof value === "string";
}

Discriminated Unions — Powerful Pattern

Common type field enables exhaustive matching
// Each variant has a literal "type" field
type Action =
    | { type: "ADD_TODO"; text: string }
    | { type: "TOGGLE_TODO"; id: number }
    | { type: "DELETE_TODO"; id: number }
    | { type: "CLEAR_COMPLETED" };

function reducer(state: State, action: Action): State {
    switch (action.type) {
        case "ADD_TODO":
            // TypeScript knows: action has `text`
            return { ...state, todos: [...state.todos, action.text] };

        case "TOGGLE_TODO":
        case "DELETE_TODO":
            // TypeScript knows: action has `id`
            return { ...state, /* ... */ };

        case "CLEAR_COMPLETED":
            // TypeScript knows: no extra properties
            return { ...state, todos: state.todos.filter(t => !t.done) };

        default:
            // Exhaustiveness check - error if we miss a case
            const _exhaustive: never = action;
            return state;
    }
}

keyof & typeof Operators

keyof — Get property keys
interface User {
    id: number;
    name: string;
    email: string;
}

// "id" | "name" | "email"
type UserKeys = keyof User;

// Type-safe property access
function getValue(
    obj: T,
    key: K
): T[K] {
    return obj[key];
}

const user: User = { id: 1, name: "Alice", email: "..." };
getValue(user, "name");  // string
getValue(user, "id");    // number
getValue(user, "foo");   // Error!
typeof — Get type from value
// Get type from a value
const config = {
    apiUrl: "https://api.example.com",
    timeout: 5000,
    retries: 3
};

// { apiUrl: string; timeout: number; retries: number }
type Config = typeof config;

// Combine with keyof
type ConfigKey = keyof typeof config;
// "apiUrl" | "timeout" | "retries"

// Great for constants
const COLORS = {
    primary: "#007bff",
    secondary: "#6c757d",
    success: "#28a745"
} as const;

type ColorName = keyof typeof COLORS;
type ColorValue = typeof COLORS[ColorName];

Click any option for details

strictNullChecks

noImplicitAny

Migrating JavaScript to TypeScript

1
Add TypeScript alongside JS

Install TS, create tsconfig.json with "allowJs": true

2
Rename files incrementally

Start with leaf files (no dependencies), rename .js to .ts

3
Fix errors, add types

Add type annotations where needed, use any temporarily if stuck

4
Enable strict mode

Turn on strict options one by one, fix remaining issues

5
Eliminate any

Replace all any with proper types

Pro tip: Use // @ts-expect-error for known issues you'll fix later. It errors if the issue is fixed!
